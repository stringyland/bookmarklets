<input> with no accessible description

← Return to main list

The purpose this test is to show the cascade order of how accessible descriptions are applied in the accessibility tree for the <input> element. The priority order from highest to lowest is:

  1. If present, use aria-describedby value.
  2. Otherwise, if present, use title value (as per the "Rabbit" example below).
  3. If none of the above are provided, there is no accessible description.

This example should not be considered proper practice, and is for testing purposes only.

Example

Code

<label for="fish">Fish</label>
<input
  type="text"
  id="fish"
  placeholder="fox"
>

Assistive technologies

VoiceOver

  • Chrome: Fish. Fox. Edit text.
  • Firefox: Fish. Edit text.
  • Safari: Fish. Fox. Edit text.

NVDA

  • Chrome: Fish. Edit. Fox. Blank.
  • Firefox: Fish. Edit. Has autcomplete. Fox. Blank.
  • Edge: Fish. Edit. Fox. Blank.

JAWS

  • Chrome: Fish. Edit. Fox. Type in text.
  • Firefox: Fish. Edit. Fox. Type in text.
  • Edge: Fish. Edit. Type in text.
