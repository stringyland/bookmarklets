This is a test case designed to determine how the <input type="datetime-local"> element is announced by various screen readers.

Example

Code

<div>
  <label for="datetime-local">Choose a date/time</label>
  <input type="datetime-local" id="datetime-local">
</div>

Assistive technologies

VoiceOver

  • Chrome: Day. Choose a date/time. Stepper. DD.
  • Firefox: Choose a date/time. Edit text.
  • Safari: Choose a date/time. Edit text.

NVDA

  • Chrome: Choose a date/time. Date edit. Day. Choose a date/time. Spin button. 0.
  • Firefox: Choose a date/time. Edit. Has autocomplete. Blank.
  • Edge: Choose a date/time. Date edit. Day. Choose a date/time. Spin button. 0.

JAWS

  • Chrome: Choose a date/time. Date time edit. Day. Choose a date/time. Spin button. 0.
  • Firefox: Choose a date/time. Edit. Type in text.
  • Edge: Choose a date/time. Date time edit. Day. Choose a date/time. Spin button. 0.
