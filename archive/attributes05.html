<input> with aria-valuemin

← Return to main list

This is a test case designed to determine how the aria-valuemin attribute is announced. In real situations, authors should always provide clear instructions on how to successfully complete form questions.

Example

Code

<div>
  <label for="number">Choose a number</label>
  <input 
    type="number"
    id="number"
    aria-valuemin="5"
  >
</div>

Assistive technologies

VoiceOver

  • Chrome: Choose a number. Stepper. [aria-valuemin attribute not announced]
  • Firefox: Choose a number. Stepper. [aria-valuemin attribute not announced]
  • Safari: Choose a number. Incrementable. Edit text. [aria-valuemin attribute not announced]

NVDA

  • Chrome: Choose a number. Spin button. Editable. [aria-valuemin attribute not announced]
  • Firefox: Choose a number. Spin button. Has autocomplete. Editable. [aria-valuemin attribute not announced]
  • Edge: Choose a number. Spin button. Editable. [aria-valuemin attribute not announced]

JAWS

  • Chrome: Choose a number. Edit. Spin box. To set the value use the arrow keys or type the value. [aria-valuemin attribute not announced]
  • Firefox: Choose a number. Edit. Spin box. To set the value use the arrow keys or type the value. [aria-valuemin attribute not announced]
  • Edge: Choose a number. Edit. Spin box. To set the value use the arrow keys or type the value. [aria-valuemin attribute not announced]
